# ระบบเช็คชื่อเข้าห้อง — คู่มือติดตั้ง

หน้าเว็บ `check-in.html` แบ่งเป็น **2 สิทธิ์**:

| สิทธิ์ | ทำอะไรได้ |
|:---|:---|
| 👩‍🎓 **นักศึกษา** (ค่าเริ่มต้น) | ดูสถานะการเช็คชื่อทั้งห้องได้อย่างเดียว (อ่านอย่างเดียว) |
| 👨‍🏫 **อาจารย์** (ใส่รหัสผ่าน) | เรียกชื่อ → กด **เช็คชื่อ / ยกเลิก** การมาเรียนของนักศึกษา |

เนื่องจากเว็บเป็น static (ไม่มี backend) จึงใช้ **Google Apps Script** เป็นตัวกลางเขียนลงชีต
และตรวจสอบรหัสผ่านอาจารย์ที่ฝั่ง backend (นักศึกษาจึงเขียนข้อมูลไม่ได้)

```
นักศึกษา ─(อ่าน)─┐
                 ├─→ Apps Script Web App ─→ Google Sheet (แท็บ Attendance)
อาจารย์ ─(เขียน)─┘   ตรวจรหัสผ่านก่อนเขียน
```

## ฐานข้อมูล

- สเปรดชีต: สร้าง Google Sheet ขึ้นมาเพื่อเก็บข้อมูลรายชื่อนักศึกษา
- **แท็บแรก** = ทะเบียนรายชื่อ ต้องมีหัวคอลัมน์ `studentId` และ `Fullname` (หรือ `Name`)
- สคริปต์จะสร้างแท็บ **`Attendance`** ให้อัตโนมัติ (คอลัมน์: Timestamp, Date, studentId, Fullname, Session)

## ขั้นตอนติดตั้ง

1. เปิดสเปรดชีต → เมนู **Extensions → Apps Script**
2. ลบโค้ดเดิมทั้งหมด แล้ววางเนื้อหาจากไฟล์ [`Code.gs`](Code.gs) → กด **Save** (💾)
3. **ตั้งรหัสผ่านอาจารย์:**
   - ในฟังก์ชัน `setTeacherKey()` แก้ข้อความ `'เปลี่ยนรหัสนี้'` เป็นรหัสที่ต้องการ
   - เลือกฟังก์ชัน `setTeacherKey` ที่แถบบน แล้วกด **Run** 1 ครั้ง (อนุญาตสิทธิ์ถ้าถูกถาม)
   - รหัสจะถูกเก็บใน **Script Properties** ไม่อยู่ในซอร์สโค้ด (จึงไม่หลุดไปกับ GitHub)
   - 🔒 หลังรันแล้ว แนะนำให้ลบข้อความรหัสจริงออกจาก `setTeacherKey()` กลับเป็นค่ากลาง ๆ
4. **Deploy:**
   - ครั้งแรก: **Deploy → New deployment → Web app** (Execute as: `Me`, Who has access: `Anyone`)
   - ถ้า deploy ไว้แล้ว/แก้โค้ดใหม่: **Manage deployments → ✏️ Edit → Version: New version → Deploy**
     (ลิงก์ `/exec` เดิมจะอัปเดต ไม่ต้องเปลี่ยน API_URL)
5. คัดลอกลิงก์ Web App URL (ลงท้ายด้วย `/exec`) ไปตั้งค่าที่ตัวแปร `API_URL` ในหน้า:
   - `check-in.md`
   - `attendance-summary.md`
   - `submission-check.md`

## ทดสอบ

- เปิด Web app URL ตรง ๆ → ควรเห็น JSON `{"ok":true,"today":...,"roster":[...]}`
- เปิดหน้า `check-in.html`:
  - มุมมองเริ่มต้น = นักศึกษา (เห็นสถานะทั้งห้อง ไม่มีปุ่ม) อัปเดตอัตโนมัติทุก 20 วินาที
  - กด **👨‍🏫 โหมดอาจารย์** → ใส่รหัสผ่าน → จะมีปุ่ม **เช็คชื่อ / ยกเลิก** ในแต่ละแถว
  - กดเช็คชื่อ → ดูแถวใหม่ในแท็บ `Attendance` ของชีต

## หน้าสรุปการมาเรียน

- หน้า `attendance-summary.html` แสดงจำนวนวันที่มาเรียนต่อคนทั้งเทอม + ร้อยละ + ตารางรายวัน
- ใช้ข้อมูลจากแท็บ `Attendance` ผ่าน `?mode=summary` ของ Web App เดียวกัน (อ่านอย่างเดียว)
- "จำนวนคาบ" คิดจากวันที่ที่มีการเช็คชื่ออย่างน้อย 1 คนในแท็บ `Attendance`

## ตรวจการส่งงาน (Submissions)

หน้า `submission-check.html` แสดงตารางว่าใครส่งงานแต่ละสัปดาห์แล้วบ้าง (เหมือนหน้าสรุปการมาเรียน)
โดยอ่านคำตอบของ Google Form ที่ลิงก์มาเก็บในสเปรดชีตเดียวกัน

**ตั้งค่าต่อ 1 ฟอร์ม (ทำครั้งเดียวต่อสัปดาห์):**

1. เปิด Google Form ของสัปดาห์นั้น → แท็บ **Responses** → ไอคอน Sheets (**Link to Sheets**)
   → เลือก **Select existing spreadsheet** → เลือกสเปรดชีตทะเบียนรายชื่อ (ตัวเดียวกับระบบเช็คชื่อ)
2. ในสเปรดชีต จะมีแท็บคำตอบใหม่เพิ่มมา → **เปลี่ยนชื่อแท็บ** เป็น `W1` (สัปดาห์ 1), `W2` (สัปดาห์ 2), …
3. ฟอร์มต้องมีคำถามที่ถาม **รหัสนักศึกษา** หรือ **ชื่อ-สกุล** อย่างน้อยหนึ่งอย่าง
   - คอลัมน์รหัส: หัวคอลัมน์มีคำว่า "รหัส" หรือ "studentId"
   - คอลัมน์ชื่อ: หัวคอลัมน์มีคำว่า "ชื่อ" หรือ "name"
   ระบบจะจับคู่กับทะเบียนด้วย **รหัสก่อน** ถ้าไม่ตรงจะลองเทียบ **ชื่อ** ต่อ

> - เทียบรหัสโดยใช้เลขส่วนหน้าก่อน `-` (เช่น `691540005001-2` ↔ `691540005001`)
> - เทียบชื่อโดยตัดคำนำหน้า (นาย/นาง/นางสาว) และช่องว่างออก จึงไม่ต้องเว้นวรรคให้ตรงเป๊ะ
>   แต่ตัวสะกดชื่อ-สกุลต้องตรงกับทะเบียน

- ไม่ต้องแก้ `Code.gs` เพิ่มต่อสัปดาห์ — แค่ลิงก์ฟอร์มและตั้งชื่อแท็บ `W{n}` ให้ถูก
- ⚠️ ต้อง **Deploy เวอร์ชันใหม่** ครั้งเดียวหลังอัปเดต `Code.gs` (เพิ่มโหมด submissions)

## หมายเหตุ

- กันเช็คซ้ำ: นักศึกษา 1 คนนับ "มาเรียน" ครั้งเดียว **ต่อวัน** (เขตเวลา Asia/Bangkok)
- ปุ่ม **ยกเลิก** จะลบบันทึกการมาเรียนของคนนั้นในวันนี้ออกจากแท็บ `Attendance`
- รหัสผ่านอาจารย์เก็บใน sessionStorage ของเบราว์เซอร์ (หายเมื่อปิดแท็บ) — ปลอดภัยพอสำหรับใช้ในห้อง
