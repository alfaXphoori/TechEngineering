## ส่วนที่ 1: ทำความรู้จัก IoT ผ่านกรณีศึกษา (20 นาที)

**IoT (Internet of Things)** คือระบบที่เชื่อมต่ออุปกรณ์ทางกายภาพเข้ากับอินเทอร์เน็ต เพื่อเก็บข้อมูล ประมวลผล และสั่งงานอัตโนมัติ

### กรณีศึกษา: ระบบตรวจสอบแรงสั่นสะเทือนของเครื่องจักร (Vibration Monitoring)

ในโรงงานผลิตชิ้นส่วนยานยนต์ วิศวกรเครื่องกลติดตั้งเซ็นเซอร์วัดแรงสั่นสะเทือน (Accelerometer) บนมอเตอร์ไฟฟ้า โดยใช้บอร์ด ESP32 อ่านค่าและส่งข้อมูลผ่าน Wi-Fi ไปยัง Dashboard บนคลาวด์ เมื่อค่าสั่นสะเทือนเกินเกณฑ์ ระบบจะแจ้งเตือนให้ซ่อมบำรุงก่อนเครื่องจักรเสียหาย

**1.** จากกรณีศึกษาข้างต้น ให้ระบุองค์ประกอบทั้ง 4 ของระบบ IoT ลงในตาราง:

| องค์ประกอบ | คำอธิบาย | สิ่งที่ใช้ในกรณีศึกษา |
|---|---|---|
| 1. อุปกรณ์/เซ็นเซอร์ (Sensor/Device) | ตัวรับรู้ข้อมูลจากสิ่งแวดล้อม | |
| 2. การเชื่อมต่อ (Connectivity) | ช่องทางส่งข้อมูล | |
| 3. การประมวลผล (Processing) | วิเคราะห์ข้อมูลที่ได้รับ | |
| 4. การแสดงผล/สั่งงาน (UI/Action) | แสดงผลหรือตอบสนอง | |

**2.** ในสาขาวิศวกรรมเครื่องกล ให้ยกตัวอย่างระบบ IoT อื่นอีก 1 ระบบ แล้วระบุองค์ประกอบทั้ง 4:

## ส่วนที่ 3: สร้างโปรเจกต์ ESP32 แรก — Blink LED (30 นาที)

### ขั้นตอนปฏิบัติ

**1.** กลับไปที่โปรเจกต์ Wokwi ที่สร้างไว้ ลบโค้ดเดิมทั้งหมด แล้วพิมพ์โค้ดด้านล่าง:

```cpp
// Lab 1: Blink LED บน ESP32
// กะพริบ LED ทุก 1 วินาที

#define LED_PIN 2  // ขา GPIO 2 (LED บนบอร์ด ESP32 DevKit)

void setup() {
  pinMode(LED_PIN, OUTPUT);    // ตั้งขา LED เป็นเอาต์พุต
  Serial.begin(115200);        // เปิด Serial Monitor ที่ 115200 baud
  Serial.println("=== ESP32 Blink Start ===");
}

void loop() {
  digitalWrite(LED_PIN, HIGH); // เปิด LED
  Serial.println("LED ON");
  delay(1000);                 // รอ 1000 มิลลิวินาที (1 วินาที)

  digitalWrite(LED_PIN, LOW);  // ปิด LED
  Serial.println("LED OFF");
  delay(1000);                 // รอ 1 วินาที
}
```

**2.** กดปุ่ม **▶ (Play)** เพื่อรัน Simulation → สังเกต LED บนบอร์ด ESP32 ที่กะพริบ

**3.** คลิกแท็บ **Serial Monitor** (ด้านล่างของ Simulation) → ตรวจสอบข้อความที่แสดง

**4.** บันทึกผลการทดลอง:

| รายการ | ผลที่สังเกตได้ |
|---|---|
| LED กะพริบหรือไม่? | |
| ข้อความใน Serial Monitor | |
| ความถี่การกะพริบโดยประมาณ | |

## ส่วนที่ 4: ต่อ LED ภายนอก + ตัวต้านทาน (35 นาที)

### ขั้นตอนปฏิบัติ

**1.** ในหน้า Simulation ของ Wokwi คลิก **"+" (Add Component)** แล้วเพิ่มอุปกรณ์:
   - **LED** (สีแดง) × 1
   - **Resistor** (ตัวต้านทาน 220Ω) × 1

**2.** ต่อวงจรตามนี้:

```
ESP32 GPIO 15 ──── ขายาว (+) LED ──── ขาสั้น (-) LED ──── Resistor 220Ω ──── GND
```

> 💡 **หมายเหตุ:** ขายาวของ LED คือขั้วบวก (Anode) ขาสั้นคือขั้วลบ (Cathode) ตัวต้านทานจำกัดกระแสไม่ให้ LED เสียหาย

**3.** แก้ไขโค้ดให้ควบคุม LED ภายนอก:

```cpp
#define LED_ONBOARD 2    // LED บนบอร์ด
#define LED_EXTERNAL 15  // LED ภายนอกที่ต่อกับ GPIO 15

void setup() {
  pinMode(LED_ONBOARD, OUTPUT);
  pinMode(LED_EXTERNAL, OUTPUT);
  Serial.begin(115200);
  Serial.println("=== External LED Test ===");
}

void loop() {
  // เปิด LED ทั้งสองพร้อมกัน
  digitalWrite(LED_ONBOARD, HIGH);
  digitalWrite(LED_EXTERNAL, HIGH);
  Serial.println("Both LEDs ON");
  delay(500);

  // ปิด LED ทั้งสองพร้อมกัน
  digitalWrite(LED_ONBOARD, LOW);
  digitalWrite(LED_EXTERNAL, LOW);
  Serial.println("Both LEDs OFF");
  delay(500);
}
```

**4.** กดรัน Simulation แล้วบันทึกผล:

| รายการ | ผลที่สังเกตได้ |
|---|---|
| LED บนบอร์ดกะพริบหรือไม่? | |
| LED ภายนอกกะพริบหรือไม่? | |
| จังหวะการกะพริบตรงกันหรือไม่? | |
