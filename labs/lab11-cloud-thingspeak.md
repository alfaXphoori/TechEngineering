# 🔬 Lab 11: เชื่อมต่อคลาวด์ ThingSpeak

## วัตถุประสงค์
- เข้าใจหลักการทำงานของการเชื่อมต่ออุปกรณ์ IoT กับแพลตฟอร์มคลาวด์ (Cloud Platform)
- สามารถสร้าง Channel และจัดการ API Key บน ThingSpeak ได้
- เขียนโปรแกรมภาษา C++ เพื่อให้บอร์ด ESP32 ส่งข้อมูลเซนเซอร์ขึ้นไปเก็บบน ThingSpeak ผ่านเครือข่าย Wi-Fi
- เข้าใจข้อจำกัดเรื่องช่วงเวลาการอัปเดตข้อมูลและสามารถอ่านค่าจากกราฟได้

## ส่วนที่ 1: การเตรียมความพร้อมบน ThingSpeak (15 นาที)
ThingSpeak เป็นแพลตฟอร์ม IoT ที่ให้บริการฟรีสำหรับการเก็บและแสดงผลข้อมูล (Data Logging) ในรูปแบบกราฟ

1. เข้าไปที่เว็บไซต์ [thingspeak.com](https://thingspeak.com/) และทำการสมัครสมาชิก (Sign Up) หรือเข้าสู่ระบบด้วยบัญชี MathWorks
2. เมื่อเข้าสู่ระบบแล้ว ให้คลิกที่ปุ่ม **New Channel** เพื่อสร้างช่องทางการรับข้อมูลใหม่
3. กรอกข้อมูลรายละเอียดของ Channel ดังนี้:
   - **Name:** การวัดอุณหภูมิและความชื้น (หรือตั้งชื่อตามต้องการ)
   - **Field 1:** Temperature (ทำเครื่องหมายถูกที่ช่อง Field 1)
   - **Field 2:** Humidity (ทำเครื่องหมายถูกที่ช่อง Field 2)
   - เลื่อนลงมาด้านล่างสุด แล้วคลิก **Save Channel**
4. ระบบจะพามาที่หน้า Channel ของเรา ให้คลิกไปที่แท็บ **API Keys**
5. จดบันทึกค่า **Channel ID** และคัดลอก **Write API Key** ของนักศึกษาเก็บไว้ (นี่คือรหัสลับที่ใช้ยืนยันตัวตนเพื่ออนุญาตให้ ESP32 ส่งข้อมูลเข้า Channel นี้ได้)

## ส่วนที่ 2: การจำลองวงจรและเขียนโปรแกรมบน Wokwi (45 นาที)
ในส่วนนี้เราจะจำลองการทำงานของ ESP32 เพื่อสร้างข้อมูลแบบสุ่ม (Random) แล้วส่งขึ้นไปยัง ThingSpeak

1. เปิดเว็บไซต์ [wokwi.com](https://wokwi.com/) และสร้างโปรเจกต์ **ESP32** ใหม่
2. ไปที่แท็บ **Library Manager** (ไอคอนรูปหนังสือบวก) ค้นหาและคลิกเพื่อติดตั้งไลบรารีชื่อ **ThingSpeak**
3. คัดลอกโค้ดด้านล่างไปวางทับในไฟล์ `sketch.ino`

```cpp
#include <WiFi.h>
#include "ThingSpeak.h"

// ตั้งค่า Wi-Fi สำหรับโปรแกรมจำลอง Wokwi (ไม่ต้องแก้ไข)
const char* ssid = "Wokwi-GUEST";
const char* password = "";

// ตั้งค่า ThingSpeak (ให้นักศึกษาแก้ไขเป็นข้อมูลของตนเอง)
unsigned long myChannelNumber = XXXXXXX; // ใส่ Channel ID ที่เป็นตัวเลข
const char * myWriteAPIKey = "XXXXXXXXXXXXXXXX"; // ใส่ Write API Key ในเครื่องหมายคำพูด

WiFiClient  client;

void setup() {
  Serial.begin(115200);
  
  // เริ่มต้นการเชื่อมต่อ Wi-Fi
  WiFi.begin(ssid, password);
  Serial.print("Connecting to Wi-Fi");
  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }
  Serial.println("\nConnected!");

  // เริ่มต้นการเชื่อมต่อไลบรารี ThingSpeak
  ThingSpeak.begin(client);
}

void loop() {
  // จำลองการอ่านค่าเซนเซอร์ (ในงานจริงสามารถเปลี่ยนเป็นการอ่านจากเซนเซอร์ DHT22 ได้)
  float temp = random(200, 350) / 10.0; // สุ่มค่าอุณหภูมิในช่วง 20.0 - 35.0 องศาเซลเซียส
  float humid = random(400, 800) / 10.0; // สุ่มค่าความชื้นในช่วง 40.0 - 80.0 %
  
  Serial.printf("Temp: %.1f C, Humidity: %.1f %%\n", temp, humid);

  // เตรียมแพ็กเกจข้อมูลเพื่อส่งเข้า Field ต่างๆ
  ThingSpeak.setField(1, temp);
  ThingSpeak.setField(2, humid);

  // ส่งข้อมูลทั้งหมดขึ้น ThingSpeak
  int httpCode = ThingSpeak.writeFields(myChannelNumber, myWriteAPIKey);
  
  // ตรวจสอบผลลัพธ์การส่งข้อมูล
  if(httpCode == 200){
    Serial.println("Channel update successful.");
  } else {
    Serial.println("Problem updating channel. HTTP error code " + String(httpCode));
  }

  // ข้อควรระวัง: ThingSpeak แบบบัญชีฟรี จำกัดการอัปเดตข้อมูลได้เร็วที่สุดทุกๆ 15 วินาที
  // หากส่งเร็วกว่านี้ข้อมูลจะถูกปฏิเสธ
  delay(15000); 
}
```

4. **จุดสำคัญ:** แก้ไขตัวแปร `myChannelNumber` และ `myWriteAPIKey` ในโค้ดให้ตรงกับข้อมูล Channel ของนักศึกษาที่จดไว้ในส่วนที่ 1
5. กดปุ่ม **Start Simulation** (รูปเล่น) สังเกตข้อความที่แสดงในแท็บ Serial Monitor
6. กลับไปที่เว็บไซต์ ThingSpeak เปิดหน้า Channel ของตนเองที่แท็บ **Private View**
7. รอดูข้อมูลที่จะถูกพล็อตเป็นกราฟแบบเรียลไทม์ (อาจต้องรอประมาณ 15-30 วินาทีจึงจะเห็นจุดแรก)
