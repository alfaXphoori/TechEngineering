## วัตถุประสงค์
*   เพื่อให้นักศึกษาสามารถส่งข้อมูลจากเซ็นเซอร์ไปจัดเก็บบนระบบคลาวด์ (Cloud Data Logging) อย่างต่อเนื่องได้
*   เพื่อเรียนรู้วิธีการดึงข้อมูลย้อนหลัง (Historical Data) ผ่าน Read API ในรูปแบบข้อมูล JSON
*   เพื่อฝึกออกแบบและตั้งค่ากฎการแจ้งเตือน (Alert Rules) เมื่อค่าเซ็นเซอร์ผิดปกติผ่านบริการของบุคคลที่สาม (เช่น ThingSpeak React, IFTTT หรือ Email)
*   เพื่อประยุกต์ใช้ระบบจัดเก็บข้อมูลและการแจ้งเตือนกับการตรวจสอบสถานะของระบบทางวิศวกรรมเครื่องกล (Condition Monitoring)

## อุปกรณ์ที่ใช้ในการทดลอง
*   โปรแกรมจำลอง Wokwi (https://wokwi.com) สำหรับจำลองการทำงานของฮาร์ดแวร์

*   บัญชีแพลตฟอร์ม IoT: ThingSpeak (https://thingspeak.com)
*   บัญชีบริการเชื่อมต่อระบบ: IFTTT (https://ifttt.com)

## ส่วนที่ 1: การจัดเก็บข้อมูลบนคลาวด์ต่อเนื่อง (Cloud Data Logging) (30 นาที)

ในส่วนนี้ เราจะจำลองการอ่านค่าอุณหภูมิจากเครื่องจักรกล (เซ็นเซอร์ DHT22) และส่งไปเก็บบนแพลตฟอร์ม ThingSpeak เพื่อให้สามารถดูกราฟความเปลี่ยนแปลงของอุณหภูมิย้อนหลังได้

1.  เปิดหน้าเว็บไซต์ ThingSpeak เข้าสู่ระบบ และสร้างช่องทางรับข้อมูล (Channel) ใหม่ ตั้งชื่อว่า "Machine_Monitor_01"
2.  เปิดใช้งานช่องรับข้อมูล (Field) จำนวน 2 ช่อง ได้แก่ Field 1 (Temperature) และ Field 2 (Humidity) แล้วกดบันทึก
3.  ไปที่แท็บ **API Keys** และคัดลอก **Write API Key** ของ Channel มาเตรียมไว้
4.  เปิดเว็บไซต์ Wokwi และสร้างโปรเจกต์สำหรับ ESP32 ใหม่
5.  เพิ่มอุปกรณ์เซ็นเซอร์ DHT22 ลงในพื้นที่ทำงาน และลากสายเชื่อมต่อขา SDA ของเซ็นเซอร์เข้ากับขา 15 ของบอร์ด ESP32
6.  เปิดแท็บ `Library Manager` (ไอคอนรูปหนังสือ) และเพิ่มไลบรารีชื่อ `DHT sensor library for ESPx`
7.  คัดลอกและวางโค้ดต่อไปนี้ลงในไฟล์ `sketch.ino` (อย่าลืมเปลี่ยน `YOUR_WRITE_API_KEY` เป็นคีย์ของคุณ)

```cpp
#include <WiFi.h>
#include <HTTPClient.h>
#include "DHTesp.h"

// ตั้งค่า WiFi ของ Wokwi
const char* ssid = "Wokwi-GUEST";
const char* password = "";

// ใส่ Write API Key ของ ThingSpeak
const char* apiKey = "YOUR_WRITE_API_KEY"; 

DHTesp dhtSensor;
const int dhtPin = 15;

void setup() {
  Serial.begin(115200);
  dhtSensor.setup(dhtPin, DHTesp::DHT22);
  
  // เริ่มต้นการเชื่อมต่อเครือข่าย WiFi
  WiFi.begin(ssid, password);
  Serial.print("Connecting to WiFi");
  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }
  Serial.println("\nWiFi Connected!");
}

void loop() {
  // อ่านค่าอุณหภูมิและความชื้น
  TempAndHumidity data = dhtSensor.getTempAndHumidity();
  
  // ตรวจสอบว่า WiFi ยังเชื่อมต่ออยู่หรือไม่
  if (WiFi.status() == WL_CONNECTED) {
    HTTPClient http;
    // สร้าง URL สำหรับส่งข้อมูลผ่าน API
    String url = "http://api.thingspeak.com/update?api_key=" + String(apiKey) + 
                 "&field1=" + String(data.temperature) + 
                 "&field2=" + String(data.humidity);
    
    http.begin(url);
    int httpCode = http.GET(); // ส่ง HTTP GET Request
    
    if (httpCode > 0) {
      Serial.print("Data sent successfully. Temp: ");
      Serial.print(data.temperature);
      Serial.println(" C");
    } else {
      Serial.println("Error sending data");
    }
    http.end();
  }
  
  // หน่วงเวลา 15 วินาที เนื่องจาก ThingSpeak รับข้อมูลได้เร็วสุดทุกๆ 15 วินาที
  delay(15000); 
}
```

8.  กดปุ่ม **Play** เพื่อจำลองการทำงาน คลิกที่ตัวเซ็นเซอร์ DHT22 บน Wokwi เพื่อปรับเปลี่ยนค่าอุณหภูมิและความชื้น
9.  เปิดหน้าต่าง ThingSpeak สังเกตการเปลี่ยนแปลงของกราฟ

**ตารางบันทึกผลลัพธ์ส่วนที่ 1:**
| เวลา (Time) | อุณหภูมิที่ปรับใน Wokwi (°C) | ค่าที่ปรากฏบนกราฟ ThingSpeak (°C) | ความชื้น (%) |
| :--- | :--- | :--- | :--- |
| ครั้งที่ 1 | | | |
| ครั้งที่ 2 | | | |
| ครั้งที่ 3 | | | |

## ส่วนที่ 2: การดึงข้อมูลย้อนหลังผ่าน Read API (JSON) (20 นาที)

แพลตฟอร์ม IoT ส่วนใหญ่จะเปิดให้ดึงข้อมูล (Fetch Data) ออกมาในรูปแบบที่คอมพิวเตอร์เข้าใจง่าย เรียกว่า JSON (JavaScript Object Notation)

1.  ในหน้า Channel ของ ThingSpeak ให้เปลี่ยนไปที่แท็บ **API Keys**
2.  ดูที่หัวข้อ **Read API Keys** ให้คัดลอก URL ในช่อง API Request ทางขวามือ (ตัวอย่างเช่น `https://api.thingspeak.com/channels/XXXX/feeds.json?api_key=YYYY&results=2`)
3.  เปิดแท็บเบราว์เซอร์ใหม่ วาง URL ที่คัดลอกมา แล้วกด Enter
4.  หน้าเว็บจะแสดงข้อมูลดิบในรูปแบบ JSON ซึ่งประกอบด้วยปีกกา `{ }` และรูปแบบ "Key": "Value"

**คำถาม:** จากข้อมูล JSON ที่ปรากฏ ให้ค้นหาและระบุค่าต่อไปนี้ (ดูจาก block สุดท้ายใน array `feeds`)
*   ค่า Field 1 (อุณหภูมิ): ........................................
*   ค่า Field 2 (ความชื้น): ........................................
*   เวลาที่บันทึกข้อมูล (created_at): ...........................................

## ส่วนที่ 3: การตั้งกฎแจ้งเตือนความผิดปกติ (Alert Rules & Triggers) (40 นาที)

ในการบำรุงรักษาเครื่องจักร (Maintenance) วิศวกรจำเป็นต้องทราบทันทีหากเครื่องจักรมีอุณหภูมิสูงเกินไป เราจะสร้างระบบส่งอีเมลแจ้งเตือนโดยอัตโนมัติ

**ขั้นตอนที่ 3.1: สร้าง Webhook และ Email ผ่าน IFTTT**
1.  ล็อกอินเข้าสู่ IFTTT (https://ifttt.com) คลิกที่เมนู **Create**
2.  ส่วน **If This** ค้นหาและเลือก `Webhooks` -> เลือก `Receive a web request` -> ตั้งชื่อ Event Name ว่า `high_temp_alert` -> กด Create trigger
3.  ส่วน **Then That** ค้นหาและเลือก `Email` -> เลือก `Send me an email`
4.  ตั้งค่าหัวข้ออีเมล (Subject) และเนื้อหา (Body) ตามต้องการ แล้วกด Create action -> Finish
5.  ไปที่หน้า My Applets -> เลือก Webhooks -> คลิกแท็บ **Documentation** เพื่อคัดลอก URL (นำหน้าด้วย `https://maker.ifttt.com/...`)

**ขั้นตอนที่ 3.2: เชื่อมโยง ThingSpeak กับ IFTTT**
1.  กลับมาที่ ThingSpeak ไปที่เมนูบนสุด **Apps** -> **ThingHTTP** -> New ThingHTTP
2.  ตั้งชื่อว่า `Trigger Email`, ในช่อง URL ให้วาง URL ที่ได้จาก IFTTT ข้อ 5, ช่อง Method เลือก `POST` -> กด Save
3.  ไปที่เมนู **Apps** -> **React** -> New React
4.  ตั้งค่าเงื่อนไขดังนี้:
    *   **Condition:** If Channel `(ชื่อ Channel ของคุณ)` Field `1 (Temperature)` is `greater than` ค่า `50`
    *   **Action:** เลือก ThingHTTP -> เลือก `Trigger Email` ที่สร้างไว้
    *   **Options:** Run action `each time condition is met`
5.  กด Save React
6.  **ทดสอบระบบ (Test Trigger):** ไปที่ Wokwi ปรับอุณหภูมิของ DHT22 ให้สูงกว่า 50°C รอให้ ESP32 ส่งข้อมูล และรอรับอีเมลแจ้งเตือน (อาจใช้เวลา 1-2 นาที)

**ตารางออกแบบกฎการแจ้งเตือน (Alert Rule Design):**
*ให้นักศึกษาฝึกออกแบบกฎการแจ้งเตือนเพิ่มเติม 1 กรณี*
| กรณี (Scenario) | เซ็นเซอร์/Field | ตัวดำเนินการ (Operator) | ค่าวิกฤต (Threshold) | การกระทำ (Action) |
| :--- | :--- | :--- | :--- | :--- |
| ตัวอย่าง: มอเตอร์โอเวอร์ฮีต | Temperature (Field 1) | > (greater than) | 50 °C | ส่งอีเมลแจ้งแผนกซ่อมบำรุง |
