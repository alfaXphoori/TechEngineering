## คำอธิบายรายวิชา

การพัฒนาของอินเทอร์เน็ตของสรรพสิ่ง (ไอโอที) ตัวรับรู้ ตัวกระทำ และการเชื่อมต่อไมโครคอนโทรลเลอร์ เทคโนโลยีไร้สายและโพรโทคอลในการสื่อสารต่าง ๆ สำหรับไอโอที เทคโนโลยีคลาวด์และการประยุกต์ใช้โพรโทคอลต่าง ๆ เทคนิคต่าง ๆ ที่ใช้ในการแสดงให้เห็นข้อมูลได้และการเชื่อมต่อกับโปรแกรมของผู้ใช้ พื้นฐานการเรียนรู้ของเครื่องจักร

> Development of internet of things (IoTs), sensor, actuator, and interfacing, micro-controller, wireless technologies and communication protocols for IoT, data visualization techniques and user program interface, basic machine learning.

## ผลลัพธ์การเรียนรู้ระดับรายวิชา (CLOs) และผลลัพธ์ระดับบทเรียน (LLOs)

### CLO1 — อธิบายหลักการและสถาปัตยกรรมของระบบ IoT ตัวรับรู้ ตัวกระทำ และไมโครคอนโทรลเลอร์ได้

- **LLO1.1** อธิบายความหมาย พัฒนาการ และประโยชน์ของอินเทอร์เน็ตของสรรพสิ่ง (IoT) ได้ (สัปดาห์ 1)
- **LLO1.2** อธิบายสถาปัตยกรรมระบบ IoT (ชั้นอุปกรณ์ ชั้นเครือข่าย ชั้นประมวลผล/คลาวด์ ชั้นแอปพลิเคชัน) ได้ (สัปดาห์ 1)
- **LLO1.3** ยกตัวอย่างการประยุกต์ใช้ IoT ในงานวิศวกรรมเครื่องกลและอุตสาหกรรมได้ (สัปดาห์ 1)
- **LLO2.1** จำแนกประเภทของเซนเซอร์และอธิบายหลักการทำงานได้ (สัปดาห์ 2)
- **LLO2.2** อธิบายคุณลักษณะของเซนเซอร์ (ช่วงวัด ความไว ความละเอียด ความคลาดเคลื่อน) ได้ (สัปดาห์ 2)
- **LLO2.3** แยกแยะสัญญาณดิจิทัลและแอนะล็อกที่ได้จากเซนเซอร์ได้ (สัปดาห์ 2)
- **LLO3.1** อธิบายประเภทและหลักการทำงานของตัวกระทำ (LED, รีเลย์, มอเตอร์, เซอร์โว) ได้ (สัปดาห์ 3)
- **LLO4.1** อธิบายสถาปัตยกรรมไมโครคอนโทรลเลอร์ ESP32 และหน้าที่ของขา GPIO ได้ (สัปดาห์ 4)
- **LLO15.1** อธิบายแนวคิดและประเภทของการเรียนรู้ของเครื่อง (supervised/unsupervised) ได้ (สัปดาห์ 15)

### CLO2 — เลือกใช้และอธิบายเทคโนโลยีไร้สาย โพรโทคอลการสื่อสาร และเทคโนโลยีคลาวด์สำหรับ IoT ได้

- **LLO7.1** อธิบายคลื่นความถี่ มาตรฐาน และหลักการของ Wi-Fi สำหรับ IoT ได้ (สัปดาห์ 7)
- **LLO8.1** เปรียบเทียบเทคโนโลยีไร้สาย BLE, Zigbee, LoRa และ NB-IoT ด้านระยะทาง พลังงาน และอัตราข้อมูลได้ (สัปดาห์ 8)
- **LLO8.2** เลือกเทคโนโลยีไร้สายให้เหมาะสมกับโจทย์การใช้งานได้ (สัปดาห์ 8)
- **LLO9.1** อธิบายสถาปัตยกรรม Client/Server, HTTP Methods และหลักการ REST API ได้ (สัปดาห์ 9)
- **LLO10.1** อธิบายรูปแบบ Publish/Subscribe ของ MQTT (Broker, Topic, QoS) ได้ (สัปดาห์ 10)
- **LLO11.1** อธิบายรูปแบบบริการคลาวด์ (IaaS/PaaS/SaaS) และองค์ประกอบของแพลตฟอร์ม IoT ได้ (สัปดาห์ 11)

### CLO3 — ออกแบบและพัฒนาระบบ IoT ที่เชื่อมต่อเซนเซอร์/ตัวกระทำ สื่อสารข้อมูล และแสดงผลผ่านโปรแกรมของผู้ใช้ได้

- **LLO3.2** ออกแบบวงจรขับโหลดด้วยทรานซิสเตอร์/รีเลย์/มอเตอร์ไดรเวอร์ได้ (สัปดาห์ 3)
- **LLO5.1** เขียนโปรแกรมใช้ Digital I/O อ่านปุ่มและขับ LED ได้ (สัปดาห์ 5)
- **LLO5.2** เขียนโปรแกรมใช้ ADC อ่านเซนเซอร์แอนะล็อกและใช้ PWM ขับโหลดได้ (สัปดาห์ 5)
- **LLO6.2** อ่านเซนเซอร์ดิจิทัลผ่าน I2C/SPI/UART และควบคุมตัวกระทำตามเงื่อนไขได้ (สัปดาห์ 6)
- **LLO9.2** ส่งข้อมูลเซนเซอร์ผ่าน HTTP/REST ในรูปแบบ JSON ได้ (สัปดาห์ 9)
- **LLO10.2** ส่งและรับข้อมูลเซนเซอร์ผ่านโพรโทคอล MQTT ได้ (สัปดาห์ 10)
- **LLO12.1** ส่งและจัดเก็บข้อมูลเซนเซอร์บนคลาวด์ (ThingsBoard Cloud) ได้ (สัปดาห์ 12)
- **LLO13.1** เลือกชนิดกราฟและออกแบบการแสดงผลข้อมูลให้เหมาะสมได้ (สัปดาห์ 13)
- **LLO14.1** ออกแบบส่วนต่อประสานผู้ใช้ (UI) สำหรับควบคุม/ติดตามระบบ IoT ได้ (สัปดาห์ 14)

### CLO4 — ปฏิบัติการสร้าง ทดสอบ และประยุกต์ใช้ระบบ IoT พร้อมการเรียนรู้ของเครื่องเบื้องต้น และทำงานเป็นทีมอย่างรับผิดชอบ

- **LLO4.2** เขียนโครงสร้างโปรแกรมและจำลองการทำงานของ ESP32 ด้วย Wokwi ได้ (สัปดาห์ 4)
- **LLO6.1** ต่อและทดสอบการสื่อสาร I2C/SPI/UART กับเซนเซอร์จริงได้ (สัปดาห์ 6)
- **LLO7.2** เชื่อมต่อ ESP32 เข้าเครือข่าย Wi-Fi และสร้าง Web Server แสดงค่าเซนเซอร์ได้ (สัปดาห์ 7)
- **LLO11.2** เชื่อมต่ออุปกรณ์ IoT เข้ากับบริการคลาวด์ได้ (สัปดาห์ 11)
- **LLO12.2** ตั้งค่าการแจ้งเตือนและดึงข้อมูลจากคลาวด์ผ่าน API ได้ (สัปดาห์ 12)
- **LLO13.2** สร้างแดชบอร์ดแสดงผลข้อมูลแบบเรียลไทม์ด้วย Node-RED ได้ (สัปดาห์ 13)
- **LLO14.2** สร้างแอปพลิเคชัน/แดชบอร์ดควบคุมอุปกรณ์ผ่าน ThingsBoard ได้ (สัปดาห์ 14)
- **LLO15.2** ฝึกและทดสอบแบบจำลองจำแนกข้อมูลเซนเซอร์ด้วย ML/TinyML เบื้องต้นได้ (สัปดาห์ 15)

> **แนวทางการจัดการเรียนรู้:** ใช้การเรียนรู้เชิงรุก (Active Learning) ผสมระหว่างการบรรยายแบบมีปฏิสัมพันธ์ การสาธิตด้วยโปรแกรมจำลอง (Wokwi) และการเรียนรู้โดยใช้โครงงาน/ใบงานเป็นฐาน (Project/Problem-Based Learning) โดยภาคทฤษฎีและภาคปฏิบัติของแต่ละสัปดาห์เชื่อมโยงกัน (Constructive Alignment) เพื่อให้ผู้เรียนได้ลงมือสร้างและทดสอบระบบ IoT จริงควบคู่กับการวิเคราะห์ข้อมูล

> **หมายเหตุเรื่อง LLO:** LLO (Lesson Learning Outcomes) คือผลลัพธ์การเรียนรู้ระดับบทเรียนรายสัปดาห์ ที่ผู้เรียน "ทำได้" เมื่อจบสัปดาห์นั้น เขียนด้วยคำกริยาที่วัดผลได้ตาม Bloom's Taxonomy และเชื่อมโยงขึ้นไปยัง CLO ที่เกี่ยวข้อง (ระบุในวงเล็บท้ายแต่ละข้อ)

## แผนการสอนรายสัปดาห์ (15 สัปดาห์)

### สัปดาห์ที่ 1 — บทนำรายวิชา (Introduction) และอินเทอร์เน็ตของสรรพสิ่ง (IoT)

**LLO**
- **LLO1.1** อธิบายความหมาย พัฒนาการ และประโยชน์ของ IoT ได้ (CLO1)
- **LLO1.2** อธิบายสถาปัตยกรรมระบบ IoT ได้ (CLO1)
- **LLO1.3** ยกตัวอย่างการประยุกต์ใช้ IoT ในงานวิศวกรรมได้ (CLO1)

- **เนื้อหาภาคทฤษฎี (2 ชม.):** แนะนำข้อตกลง โครงสร้างรายวิชา และเกณฑ์การให้คะแนน (Syllabus Intro) · ความหมายและพัฒนาการของ IoT · สถาปัตยกรรมระบบ IoT 4 ชั้น · องค์ประกอบ (อุปกรณ์ เครือข่าย คลาวด์ แอปพลิเคชัน) · การประยุกต์ในงานวิศวกรรม
- **ปฏิบัติการ/ใบงาน (2 ชม.):** ใบงานที่ 1 รู้จัก IoT และเครื่องมือ · สมัครและใช้งาน Wokwi/Arduino IDE · ต่อวงจร LED จำลองตัวแรก
- **กิจกรรม/สื่อ:** บรรยายแบบมีปฏิสัมพันธ์ + กรณีศึกษา + สาธิตเครื่องมือ — สไลด์ เอกสารประกอบ LMS
- **ศึกษาด้วยตนเอง (5 ชม.):** ทบทวนสถาปัตยกรรม IoT สืบค้นตัวอย่างการประยุกต์ใช้งานจริง

### สัปดาห์ที่ 2 — การเชื่อมต่อสัญญาณดิจิทัลและแอนะล็อก

**LLO**
- **LLO2.1** เขียนโปรแกรมใช้ Digital I/O อ่านปุ่มและขับ LED ได้ (CLO3)
- **LLO2.2** เขียนโปรแกรมใช้ ADC อ่านเซนเซอร์แอนะล็อกและใช้ PWM ขับโหลดได้ (CLO3, CLO4)

- **เนื้อหาภาคทฤษฎี (2 ชม.):** Digital I/O และ pull-up/pull-down · ADC ความละเอียดและการแปลงค่า · PWM และ duty cycle · การปรับขนาดสัญญาณ (scaling/mapping)
- **ปฏิบัติการ/ใบงาน (2 ชม.):** ใบงานที่ 5 ADC และ PWM · อ่านค่า LDR/โพเทนชิออมิเตอร์ · หรี่ไฟ LED และควบคุมความเร็วมอเตอร์ด้วย PWM
- **กิจกรรม/สื่อ:** บรรยาย + การทดลองและวัดสัญญาณ — Wokwi, ชุดทดลอง
- **ศึกษาด้วยตนเอง (5 ชม.):** ฝึกแปลงค่า ADC เป็นปริมาณจริง ทำแบบฝึกหัด PWM

### สัปดาห์ที่ 3 — ตัวรับรู้ (Sensors)

**LLO**
- **LLO3.1** จำแนกประเภทของเซนเซอร์และอธิบายหลักการทำงานได้ (CLO1)
- **LLO3.2** อธิบายคุณลักษณะของเซนเซอร์ได้ (CLO1)
- **LLO3.3** แยกแยะสัญญาณดิจิทัลและแอนะล็อกจากเซนเซอร์ได้ (CLO1)

- **เนื้อหาภาคทฤษฎี (2 ชม.):** ประเภทเซนเซอร์ (อุณหภูมิ ความชื้น แสง ระยะทาง การเคลื่อนไหว แก๊ส) · หลักการแปลงปริมาณทางกายภาพเป็นสัญญาณไฟฟ้า · คุณลักษณะเซนเซอร์ · สัญญาณดิจิทัล/แอนะล็อก
- **ปฏิบัติการ/ใบงาน (2 ชม.):** ใบงานที่ 2 อ่านค่าเซนเซอร์ดิจิทัล · ทดลองปุ่มกด/PIR · สังเกตสถานะ HIGH/LOW
- **กิจกรรม/สื่อ:** บรรยาย + สาธิตเซนเซอร์ + การทดลอง — ชุดทดลอง โปรแกรมจำลอง
- **ศึกษาด้วยตนเอง (5 ชม.):** อ่านดาต้าชีตเซนเซอร์ เปรียบเทียบคุณลักษณะเซนเซอร์ชนิดต่าง ๆ

### สัปดาห์ที่ 4 — ตัวกระทำและการเชื่อมต่อ (Actuators & Interfacing)

**LLO**
- **LLO4.1** อธิบายประเภทและหลักการทำงานของตัวกระทำได้ (CLO1)
- **LLO4.2** ออกแบบวงจรขับโหลดด้วยทรานซิสเตอร์/รีเลย์/มอเตอร์ไดรเวอร์ได้ (CLO3)

- **เนื้อหาภาคทฤษฎี (2 ชม.):** ประเภทตัวกระทำ (LED, บัซเซอร์, รีเลย์, มอเตอร์ DC, เซอร์โว, สเต็ปเปอร์) · การขับโหลดด้วยทรานซิสเตอร์/MOSFET/รีเลย์ · ไดโอดกันไฟย้อน · การแยกกราวด์และแหล่งจ่าย
- **ปฏิบัติการ/ใบงาน (2 ชม.):** ใบงานที่ 3 ควบคุมตัวกระทำ · ขับ LED/รีเลย์/เซอร์โว · ทดสอบวงจรขับโหลด
- **กิจกรรม/สื่อ:** บรรยาย + สาธิต + การทดลองเป็นทีม — ชุดทดลอง โปรแกรมจำลอง
- **ศึกษาด้วยตนเอง (5 ชม.):** ออกแบบวงจรขับโหลดตามโจทย์ อ่านล่วงหน้าเรื่องไมโครคอนโทรลเลอร์

### สัปดาห์ที่ 5 — ไมโครคอนโทรลเลอร์เบื้องต้น (ESP32/Arduino)

**LLO**
- **LLO5.1** อธิบายสถาปัตยกรรม ESP32 และหน้าที่ของขา GPIO ได้ (CLO1)
- **LLO5.2** เขียนโครงสร้างโปรแกรมและจำลองการทำงานด้วย Wokwi ได้ (CLO4)

- **เนื้อหาภาคทฤษฎี (2 ชม.):** สถาปัตยกรรมไมโครคอนโทรลเลอร์ · ESP32 และความสามารถ (Wi-Fi/BLE ในตัว) · GPIO และฟังก์ชันของขา · โครงสร้างโปรแกรม Arduino (setup/loop)
- **ปฏิบัติการ/ใบงาน (2 ชม.):** ใบงานที่ 4 GPIO เบื้องต้น · เขียนโปรแกรม Blink และอ่านปุ่ม · จำลองด้วย Wokwi
